## Funcionalidad de SOLID 
En la semana 1 se realizaron varias actividad un poco variadas en focadas a python profesional y los principios SOLID.
se vieron temas como funciones puras, maquinas de estados y lo mas importantes los principios SOLID.

A lo largo de este proyecto, apliqué los cinco principios para transformar un script monolítico en un sistema modular:
### 1. S - Principio de Responsabilidad Única (SRP)
Una clase debe tener una y una sola razon para cambiar.quiere decir que debe encargarse de una unica tarea 

### 2. O - Principio de Abierto/Cerrado (OCP)
Esto quiere decir que deberia poder añadir nuevas funciones a un programador, sin necesidad de alterar el codigo original.  

### 3. L - Principio de Sustitución de Liskov (LSP)
Si hay una funcion que espera recibir un objeto de la clase padre, esa misma funcion debe operar perfectamente si le pasa cualquier objeto de sus clases hijas.

### 4. I - Principio de Segregación de la Interfaz (ISP)
Un cliente nunca debe verse obligado aimplementar una interfaz que no utilizar, ni dependerse de metodos que no usa.

### 5. D - Principio de Inversión de Dependencias (DIP)
Establece dos reglas clave para el software: los módulos de alto nivel no deben depender de los de bajo nivel (ambos deben depender de abstracciones), y las abstracciones no deben depender de los detalles (los detalles deben depender de las abstracciones.

---realizando las actividades de SOLID fui entendiendo la importancia de los principios SOLID, ya que ellos nos apoyan a un mejor entendimiento del codigo una mejor estructura de la misma asi reduciendo fallos, te permite incorporar nuevos requisitos y caracteristicas sin que el sistema colapse o se rompa y reduce la dependencia entre las distintas partes del código, lo que facilita realizar pruebas automatizadas y reutilizar componentes de manera eficiente.---------
